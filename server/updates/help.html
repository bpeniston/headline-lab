GE360 automated updates — how it works

← Back to today's updates

Every morning at 5 a.m., a script running on an always-on Mac with a blotchy screen in Brad's basement looks at the past 30 days of Google Analytics data for each publication and identifies the topics that are driving the most traffic — weighted toward recent activity so today's news beats last month's evergreen. It then logs into the Athena CMS and updates each publication's Trending Topics module with the top seven, replacing whatever was there the day before. A second script runs at 5:30 a.m. and does the same thing for Earthbox slots, swapping in the six articles with the highest recent pageview scores. Sponsored slots in both modules are detected automatically and left untouched.

Once each script finishes, it sends a summary to the publication's Slack channel showing what changed (new items highlighted) and what stayed the same. If something goes wrong — a CMS login has expired, an article can't be found, a network request times out — the script sends a Problem alert to Slack instead, with details. This page collects all of those daily summaries in one place so you can see the full picture across all five publications at a glance.

The system currently runs automatically for Defense One and Washington Technology. GovExec, Nextgov, and Route Fifty are configured and ready, if their editors want auto-updates. All five publications share the same CMS login and the same underlying scripts — adding a pub is a one-line change in a Google Sheet.
