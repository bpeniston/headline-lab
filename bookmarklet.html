How To Install
  1. Show your bookmarks bar if it's hidden — in Chrome: Ctrl+Shift+B (Win) or ⌘+Shift+B (Mac). In Firefox: View → Toolbars → Bookmarks Toolbar.
  2. Drag the red button above into your bookmarks bar. You'll see it appear as "📰 Headline Lab".
  3. That's it. Click the bookmark on any Athena post editing page or any live article on one of the five supported sites — the tool opens in a new tab with the content already loaded.
What It Does

The bookmarklet detects which kind of page you're on and reads accordingly:

  1. Athena CMS post editor — Reads the Content field via the CKEditor API, stripping HTML tags to give the AI clean plain text. Also grabs the Title and Subhed fields for extra context. Falls back to reading the raw textarea if CKEditor hasn't initialized yet.
  2. Live article pages on defenseone.com, govexec.com, nextgov.com, route-fifty.com, or washingtontechnology.com — Reads the headline (h1), deck (h2), and all body paragraphs from the article. Caption and photo credit text is automatically excluded. Useful for re-optimizing a published headline, generating social posts for an older story, or working on a colleague's piece.
  3. Everything is POSTed directly to the server — no URL length limits, full articles supported. If nothing readable is found, the tool opens with an alert so you can paste manually.
Troubleshooting
  • The bookmarklet opens Headline Lab but the text field is empty.
    Make sure you're on an Athena post editing page (URL contains /athena/post_manager/post/) or a live article page on one of the five supported sites. On Athena, also make sure the post has some body text already entered — the bookmarklet can't read a blank Content field.
  • It only grabbed the title, not the article body.
    On Athena, the CKEditor instance may not have loaded yet. Wait a second after the page finishes loading, then try again. If it still fails, let me know — I can tune the selector.
  • It grabbed the wrong text (navigation, sidebar, etc.).
    The auto-detection isn't perfect for every page layout. Send me a note with which page you were on and I can tune the selector.
  • Can I see the bookmarklet code?
    Yes — right-click the bookmark in your bar → Edit, and the full JavaScript is visible. It's self-contained, runs only in your browser, and sends text only to prefill.php on this server, which immediately hands it to the tool and discards it.
